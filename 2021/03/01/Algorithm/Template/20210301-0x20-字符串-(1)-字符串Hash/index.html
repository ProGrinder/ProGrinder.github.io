字符串Hash

字符串Hash有很多种方式，最常用的一种就是字符串前缀哈希。

字符串前缀哈希

字符串前缀哈希是一种常用的字符串匹配算法，它可以在 O(1)O(1)O(1) 内计算出一个字符串的前缀子串的哈希值，从而实现快速匹配，其原理如下：

假设我们有一个字符串 sss，其长度为 nnn，哈希函数为 h(x)h(x)h(x)，
取一固定值PPP，把字符串看作PPP进制数，取一固定值MMM，求出该PPP进制数对MMM的余数，作为字符串Hash值：

h(s[0…k])=(∑i=0ks[i]×Pi) mod Mh(s[0 \ldots k])=(\sum_{i=0}^k s[i] \times P^i )\ mod \ Mh(s[0…k])=(i=0∑k​s[i]×Pi) mod M

其中PPP 通常取一个大于 nnn 的质数，常用质数有 P=131P=131P=131 或 P=13331P=13331P=13331，此时字符串哈希冲突概率极低。
其中MMM通常取M=264M = 2^{64}M=264，即直接使用 unsigned long long 类型存储这个Hash值，溢出时自动对2642^{64}264取模。

个人认为PPP取131是因为ASCIIASCIIASCII码表一共有128个字符，所以找了一个比128大的质数。

对于该字符串sss的任意子串s[l,r]s[l, r]s[l,r]，其Hash值可以在O(1)O(1)O(1)内求出：

h(s[l,r])=s[r]−s[l−1]∗pr−l+1h(s[l, r]) = s[r] - s[l -1] * p^{r - l + 1}h(s[l,r])=s[r]−s[l−1]∗pr−l+1

总的来说，比较像前缀和的思想，因此叫字符串前缀哈希。

#include <string>
#include <vector>
#include <iostream>

using namespace std;

typedef unsigned long long ULL;

const int N = 1e6 + 7;
const int P = 131; // const int P = 13331;

vector<ULL> P(N); // P次幂

vector<ULL> get_prefix_hash(const string& s) {
	int n = s.size();
	vector<ULL> h(n + 1);
	h[0] = 0;
    p[0] = 1;
	for (int i = 1; i <= n; i++) {
		h[i] = h[i - 1] * P + s[i - 1];
        p[i] = p[i - 1] * P;
	}
	return h;
}

ULL get_substr_hash(const vector<ULL>& h, int l, int r) {
	return h[r] - h[l - 1] * p[r - l + 1];
}

int main() {
	string s = "violet violence"; 
	vector<ULL> h = get_prefix_hash(s);

	cout << "Prefix Hash of " << s << ":" << endl;	
	for (int i = 0; i < s.size(); i ++) {
		cout << s.substr(0, i + 1) << " : " << h[i + 1] << endl;
	}

	cout << endl;
	// 计算violet的'viole' 与 violence的'viole'的Hash值是否相同
	cout << get_substr_hash(h, 1, 5) << " and " << get_substr_hash(h, 8, 12) << endl;

	return 0;
}

复杂度分析

  • 时间复杂度：O(n){O(n)}O(n)， 预处理前缀和需要O(n)O(n)O(n)，查询字串的时间复杂度是常数级O(1)O(1)O(1)。
  • 空间复杂度：O(N){O(N)}O(N)，要花O(n+1)O(n + 1)O(n+1)的空间存储Hash前缀和，花O(N)O(N)O(N)的空间存储P的NNN次幂。

求PPP的次幂也可以用一个快速幂fast_pow()，用时间换空间，但这样每次查询就是O(log⁡(r−l+1))O(\log(r - l + 1))O(log(r−l+1))的了。

总之，不要使用<cmath>中自带的pow()，其返回类型是double，而浮点型转为整型会有精度损失。

#include <string>
#include <vector>
#include <iostream>

using namespace std;

typedef unsigned long long ULL;

const int P = 131; // const int P = 13331;

ULL fast_pow(ULL x, ULL n) {
    ULL res = 1;
    for (; n; n >>= 1) {
        if (n & 1) {
            res = res * x;
        }
        x = x * x;
    }  
    return res;
}

vector<ULL> get_prefix_hash(const string& s) {
	int n = s.size();
	vector<ULL> h(n + 1);
	h[0] = 0;
	for (int i = 1; i <= n; i++) {
		h[i] = h[i - 1] * P + s[i - 1];
	}
	return h;
}

ULL get_substr_hash(const vector<ULL>& h, int l, int r) {
	return h[r] - h[l - 1] * fast_pow(P, r - l + 1); // 不要使用(ULL)pow(P, r - l + 1); 
}

int main() {
	string s = "violet violence"; 
	vector<ULL> h = get_prefix_hash(s);

	cout << "Prefix Hash of " << s << ":" << endl;	
	for (int i = 0; i < s.size(); i ++) {
		cout << s.substr(0, i + 1) << " : " << h[i + 1] << endl;
	}

	cout << endl;
	// 计算violet的'viole' 与 violence的'viole'的Hash值是否相同
	cout << get_substr_hash(h, 1, 5) << " and " << get_substr_hash(h, 8, 12) << endl;

	return 0;
}
  • 时间复杂度：O(n){O(n)}O(n)， 预处理前缀和需要O(n)O(n)O(n)，查询字串的时间复杂度是常数级O(r−l+1)O(r - l + 1)O(r−l+1)。
  • 空间复杂度：O(n){O(n)}O(n)，要花O(n+1)O(n + 1)O(n+1)的空间存储Hash前缀和。

std::hash<std::string>

std::hash<std::string> 是 C++ 标准库中提供的一种哈希函数，用于将 std::string 类型的字符串映射为一个无符号整数，可以用于标准库的哈希容器（例如 unordered_map）中作为 key 的类型。

在实现上，std::hash<std::string> 采用了基于 Horner 算法的哈希算法，其基本思路是将字符串视为一个进制数，每次遍历字符串时，将前面已经计算出的结果乘以一个固定的质数（通常选取 31），然后加上当前字符的 ASCIIASCIIASCII 码值，最后得到的值就是字符串的哈希值，可以说其思想与字符串前缀哈希的思想是差不多的。std::hash<std::string> 的实现方式类似于下面的代码：

std::size_t operator()(const std::string& s) const {
    std::size_t hash = 0;
    for (char c : s) {
        hash = hash * 31 + c;
    }
    return hash;
}
