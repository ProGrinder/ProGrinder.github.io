0x40进阶数据结构-(3)-树状数组

树状数组

树状数组（Binary Index Tree / Fenwick Tree）是一种简单的数据结构，主要用来维护序列的前缀和。
它根据任意正整数关于2的不重复次幂的唯一分解性质，把区间二进制分解，支持在O(log⁡n)O(\log n)O(logn)对原序列单点更新，同时支持在O(log⁡n)O(\log n)O(logn)内查询前缀和。

任意正整数关于2的不重复次幂的唯一分解：

就是说：如果我们想查询区间[1, x]，可以先把x进行二进制分解：
假设正整数x的二进制表示为ak−1ak−2…a2a1a0a_{k - 1}a_{k - 2}\ldots a_2a_1a_0ak−1​ak−2​…a2​a1​a0​，其中等于1的位是{ai1,ai2,…aim}{\{a_{i_{1}}, a_{i_{2}}, \ldots a_{i_{m}}\}}{ai1​​,ai2​​,…aim​​}，则正整数x可以被二进制分解成：

x=2i1+2i2+…+2imx = 2^{i_1} + 2^{i_2} + \ldots + 2^{i_m}x=2i1​+2i2​+…+2im​

那么查询区间[1,x]，可以分解成查询以下log(x)个子区间：

  • 长度为2i12^{i_1}2i1​的子区间[1,2i1][1, 2^{i_1}][1,2i1​]

  • 长度为2i22^{i_2}2i2​的子区间[2i1+1,2i1+2i2][2^{i_1} + 1, 2^{i_1} + 2^{i_2}][2i1​+1,2i1​+2i2​]

  • …

  • 长度为2im2^{i_m}2im​的子区间[2i1+2i2+…+2im−1+1,2i1+2i2+…+2im][2^{i_1} + 2^{i_2} + \ldots + 2^{i_{m-1}} + 1, 2^{i_1} + 2^{i_2} + \ldots + 2^{i_m}][2i1​+2i2​+…+2im−1​+1,2i1​+2i2​+…+2im​]

这些子区间的共同特点是:若区间结尾为R，则区间长度就等于R的二进制分解下最小的2的次幂，即lowbit(R)lowbit(R)lowbit(R)

对于一个长度为N的序列aaa，我们建立一个同样长度为N的树状数组treetreetree，其中tree[x]tree[x]tree[x]保存序列aaa的区间[x−lowbit(x)+1,x][x - lowbit(x) + 1, x][x−lowbit(x)+1,x]中所有数的和，即∑i=x−lowbit(x)+1xa[i]\sum_{i=x-lowbit(x)+1}^{x}a[i]∑i=x−lowbit(x)+1x​a[i]。树状数组结构如下图所示：

该结构满足以下性质：

  1. 每个内部节点tree[x]tree[x]tree[x]保存以它为根的子树中所有节点的和
  2. 每个内部节点tree[x]tree[x]tree[x]的子节点个数等于lowbit(x)lowbit(x)lowbit(x)的位数
  3. 除树根外，每个内部节点tree[x]tree[x]tree[x]的父节点是tree[x+lowbits(x)]tree[x + lowbits(x)]tree[x+lowbits(x)]
  4. 树的深度是O(log⁡2N)O(\log_2N)O(log2​N)

单点更新(update)

单点更新就是把原序列中的一个数a[x]a[x]a[x]加上一个Δu\Delta uΔu，同时正确维护树状数组所记录的区间和。

根据上面给出的树状数组树形结构和它的性质，不难发现只有tree[x]tree[x]tree[x]及其所有祖先节点保存的区间和包含a[x]a[x]a[x]，
因此就像“爬树”一样，从tree[x]tree[x]tree[x]开始每次下标+lowbit(x)+lowbit(x)+lowbit(x)寻找到祖先节点，逐一进行更新即可。

查询前缀和(query)

查询前缀和，即序列aaa中第1∼x1 \sim x1∼x个数的和。

按照我们上述提出的方法，应该求出x的二进制表示的每个等于1的位，把[1,x][1, x][1,x]分为O(log⁡N)O(\log N)O(logN)个不重复子区间，而每个子区间的区间和都已经保存在树状数组treetreetree中，把这些子区间的区间和加起来就能在O(log⁡N)O(\log N)O(logN) 的时间内查询前缀和。

1-based树状数组板子如下：

#include <iostream>
#include <algorithm>

using namespace std;

const int N = 1e5 + 7;

struct BinaryIndexTree {
    int tr[N];
   	int lowbit(int x) {
        return x & -x;
    }
    void update(int x, int d) {
        for (int i = x; i <= n; i += lowbit(i)) tr[i] += d;
    }
    int query(int x) {
        int res = 0;
        for (int i = x; i; i -= lowbit(i)) sum += tr[i];
        return res;
    }
};

int main() {
    cin >> n;
    int a[N];
    BinaryIndexTree bit;
    memset(bit.tr, 0, sizeof bit.tr);
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        bit.update(i, a[i]); // 构建树状数组
    }
    int l, r;
    cin >> l >> r;
    cout << bit.query(r) - bit.query(l - 1); //查询区间和[l, r]
}

个人认为树状数组是要比ST表好理解不少的。

复杂度分析

  • 时间复杂度：
    • 构建：O(nlog⁡n)O(n \log n)O(nlogn)，构建相当于是做n次单点更新
    • 单点更新：O(log⁡n)O(\log n)O(logn)
    • 查询区间和：O(log⁡n)O(\log n)O(logn)
  • 空间复杂度：O(n)O(n)O(n)
